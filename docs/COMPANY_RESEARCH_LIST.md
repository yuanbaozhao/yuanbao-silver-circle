# 企业数据库首批研究清单

> 本清单用于安排企业数据库 V1 的研究顺序，不代表名单中的主体已经完成审核，也不代表对企业市场地位、经营状态或业务规模的背书。所有主体均从 `Candidate` 状态开始，品牌名、工商主体、业务边界和当前经营状态需逐一核验。

## 一、选择原则

首批 30 个研究对象优先满足以下一项或多项：

1. 能代表一种典型银发经济商业模式；
2. 与养老服务、医疗照护、智慧养老、旅居、金融保险或人生后服务存在明确关联；
3. 具有可获得的官网、年报、政府信息或主流媒体来源；
4. 能与现有 Silver Wiki 词条建立关系；
5. 对创业者、投资人或产业从业者具有研究价值。

## 三、研究任务卡

每个 Candidate 必须完成以下任务后，才可升级为 `Draft`：

- [ ] 确认品牌名称与工商主体
- [ ] 确认官方网站或权威主体页面
- [ ] 确认当前主要业务
- [ ] 确认总部或主要运营地区
- [ ] 收集不少于 3 个来源，其中至少 1 个为 A 级来源
- [ ] 建立至少 1 个 Silver Wiki 关联词条
- [ ] 标记动态数据的事实有效日期
- [ ] 区分企业事实与元宝 AI 判断

升级为 `Reviewed` 前还需完成：

- [ ] 主体、成立时间、业务范围等关键事实审校
- [ ] 重要规模数据注明口径与截至日期
- [ ] 商业模式分析有事实依据
- [ ] 知识图谱关系保存证据与置信度
- [ ] 页面不存在“待核验”占位文本
- [ ] 完成版本记录和最近审校日期

## 四、首批三家研究顺序

### 1. 泰康之家

重点研究：养老社区模式、保险支付协同、医养结合、长期资本与品牌信任。

### 2. 福寿康

重点研究：居家照护、长期护理保险支付、城市复制、护理人员组织与服务密度。

### 3. 九如城

重点研究：机构与社区服务网络、政企合作、区域运营、连锁复制与多业务协同。

## 五、发布边界

- 研究清单可使用常见品牌名，但正式档案必须以核验后的主体信息为准。
- 业务已经调整、停止或并入其他主体时，应进入 `Archived` 或明确历史时间范围。
- 集团业务与子品牌业务必须分别建档或清楚说明关系。
- 没有可靠公开来源的营收、床位、门店、用户和市场份额数字不公开。
- “竞争关系”必须谨慎使用；仅有业务相似时优先标记为 `COMPARABLE_TO`。
